# AI 顾问悬浮栏产品设计文档

## 1. 需求说明

### 1.1 需求背景
将首页的 AI 顾问入口改成悬浮栏形式，参考微信/客服小程序的悬浮球效果，提供更便捷的访问体验。

### 1.2 核心需求
| 需求 | 描述 |
|------|------|
| 位置 | 固定在页面右下角 |
| 收起状态 | 显示为小图标，默认在右下角 |
| 展开状态 | 点击后弹出悬浮面板，可进行对话 |
| 功能 | AI 顾问对话功能保持不变 |

### 1.3 业务目标
- **降低访问门槛**：用户无需跳转页面即可使用 AI 顾问
- **提升用户体验**：类似微信浮窗的交互，用户熟悉度高
- **全局可访问**：在任意页面都能快速唤起 AI 顾问

## 2. UI/交互设计

### 2.1 收起状态（悬浮球）

| 设计项 | 规格 |
|--------|------|
| **图标** | 🤖 或专用 Logo |
| **尺寸** | 56px × 56px（移动端 48px × 48px） |
| **位置** | 固定于页面右下角，距底部 24px，距右侧 24px |
| **背景** | 主色调渐变（#667eea → #764ba2）或纯色 |
| **阴影** | `0 4px 16px rgba(0,0,0,0.15)` |
| **动画** | 悬停时轻微上浮（transform: translateY(-4px)） |
| **圆角** | 50%（圆形）或 16px（圆角方形） |

**可选：消息提醒**
- 未读消息数角标（红色圆点 + 数字）
- 新消息时呼吸动画

### 2.2 展开状态（悬浮面板）

| 设计项 | 规格 |
|--------|------|
| **尺寸** | 宽度 400px，高度 600px（移动端 100% 宽度，80% 高度） |
| **位置** | 固定于页面右侧，垂直居中，距右侧 24px |
| **背景** | #ffffff |
| **边框** | `1px solid #e8e8e8` |
| **圆角** | 16px |
| **阴影** | `0 8px 32px rgba(0,0,0,0.12)` |
| **关闭方式** | 点击右上角 × 按钮 / 点击遮罩层 / ESC 键 |

**展开动画**：
- 从悬浮球位置展开（scale + opacity）
- 时长 300ms，ease-out

### 2.3 面板内部布局

```
┌─────────────────────────────────┐
│  🤖 AI 顾问           ✕        │  ← 头部（56px）
├─────────────────────────────────┤
│                                 │
│  [对话消息区域]                  │  ← 消息区（flex: 1）
│                                 │
│                                 │
├─────────────────────────────────┤
│  [输入框...] [发送]             │  ← 底部（60px）
└─────────────────────────────────┘
```

**详细分区**：

1. **头部 (56px)**
   - 左侧：🤖 图标 + "AI 顾问" 标题
   - 右侧：最小化按钮（可收起为悬浮球）

2. **消息区 (flex: 1, overflow-y: auto)**
   - 与现有 AIChat.vue 保持一致的样式
   - 用户消息：右对齐，蓝色背景
   - AI 消息：左对齐，灰色背景
   - 支持产品卡片、方案推荐等复杂消息

3. **输入区 (60px)**
   - 输入框：圆角，支持 placeholder
   - 发送按钮：主色调

### 2.4 交互流程

```
用户浏览页面
     ↓
    [点击悬浮球] ──→ 展开对话面板
     ↓                    ↓
  [点击空白区域]      [输入消息]
     ↓                    ↓
  [面板保持展开]      [发送消息]
     ↓                    ↓
  [点击最小化] ──→  收起为悬浮球
     ↓
  [页面滚动时] ──→ 悬浮球保持固定位置
```

### 2.5 页面显示策略

| 场景 | 显示悬浮栏 |
|------|------------|
| 首页 / | ✅ 显示 |
| AI 聊天页 /ai-chat | ❌ 不显示（已是完整页面） |
| 其他页面 | ✅ 建议显示 |

**说明**：
- `/ai-chat` 页面不需要显示悬浮栏（避免重复）
- 其他页面（如产品列表、试用工作台等）均显示悬浮栏
- 可以在路由守卫中控制显示逻辑

## 3. 技术实现建议

### 3.1 组件设计

```
components/
├── AIFloatButton.vue      # 悬浮球组件（收起状态）
├── AIFloatPanel.vue       # 对话面板组件（展开状态）
└── AIFloatingBar.vue      # 组合组件（父容器）
```

**AIFloatingBar.vue** 作为全局组件，在 `App.vue` 中引入：

```vue
<!-- App.vue -->
<template>
  <router-view />
  <AIFloatingBar />
</template>
```

### 3.2 状态管理

使用 Vue `reactive` 或 Pinia 管理状态：

```javascript
// 状态
const isExpanded = ref(false)     // 面板展开/收起
const hasUnread = ref(false)       // 未读消息
const messages = ref([])          // 对话历史（可选：与 /ai-chat 共享）
```

### 3.3 消息同步方案

| 方案 | 描述 | 复杂度 |
|------|------|--------|
| **方案A：独立会话** | 悬浮栏和 /ai-chat 使用独立会话 | 低 |
| **方案B：共享状态** | 使用 localStorage/Vuex 共享对话历史 | 中 |
| **方案C：同一组件** | 将 AIChat.vue 抽取为可复用的 ChatComponent | 高 |

**推荐**：方案A（独立会话），简单稳定，后续可升级为方案B

### 3.4 样式隔离

- 使用 `position: fixed` 固定定位
- 设置 `z-index: 9999` 确保在最顶层
- 移动端使用媒体查询调整尺寸

### 3.5 路由控制

在 `AIFloatingBar` 组件中：

```javascript
import { useRoute } from 'vue-router'

const route = useRoute()
const showFloatBar = computed(() => route.path !== '/ai-chat')
```

## 4. 页面展示规则

```javascript
// 伪代码
const excludedRoutes = ['/ai-chat']

const shouldShow = !excludedRoutes.includes(route.path)
```

**完整页面（不显示悬浮栏）**：
- `/ai-chat` - 已经是完整的 AI 聊天页面

**所有其他页面（显示悬浮栏）**：
- `/` - 首页
- `/trial` - 试用工作台
- `/products` - 产品列表（未来）
- `/products/:id` - 产品详情（未来）

## 5. 后续迭代建议

1. **消息持久化**：使用 localStorage 保存对话历史
2. **多会话支持**：支持开启新会话
3. **快捷入口**：侧边栏显示常用问题（参考现有 quickQuestions）
4. **通知角标**：有新消息时显示未读数
5. **移动端适配**：底部 tab 样式或全屏面板
